# Exam Flow

고등학교 정기시험 운영에 필요한 출력물을 생성하는 **클라이언트 전용** 웹 프로그램입니다.

## 프로그램 목적

나이스 학생편성현황과 학교 시험 규칙을 바탕으로 다음 출력물을 준비합니다.

- 좌석배치도
- 응시현황표 / 결시현황표
- 학생 개인별 시험 시간표
- 선택과목 응시 학생 현황표
- 시험실배정현황 / 시험실운영현황 (평가 담당자용)

기존 엑셀/VBA 기반 「고등학교 지필평가 운영 프로그램」의 데이터 흐름을 웹앱으로 재구성한 초안입니다.

## 사용 순서

1. **Step 1. 시험 규칙 설정**
   - 학년도, 학기, 차수, 고사명, 시험 일수·교시, 일차별 날짜 입력
   - 학년별 시험 시간표 입력 (같은 교시 복수 과목은 `/` 구분)
   - 학년별 이동 방식, 좌석 배치 기본값, 고사실 목록 설정

2. **Step 2. 학생편성현황 업로드**
   - 나이스에서 받은 학생편성현황 엑셀을 학년별로 업로드
   - 학생·과목 데이터 자동 생성 및 요약 확인

3. **Step 3. 시험실·좌석 배정**
   - 「시험 일정·응시자 생성」으로 시간표와 학생 데이터 연결
   - 교시별 응시자를 고사실에 배정 (정원 초과 경고)
   - 「좌석번호 자동 부여」 (학년→반→번호 순)

4. **Step 4. 배치자료 검토**
   - 시험실·좌석번호·결시정보 최종 확인 및 수정

5. **Step 5. 출력물 생성**
   - 시험 운영 현황 대시보드 확인
   - 검증 실행 후 7종 출력물 미리보기·인쇄

### 작업 저장

- **작업 저장**: 브라우저 localStorage에 저장
- **JSON 백업**: 파일로 내려받기/불러오기
- **전체 초기화**: 모든 데이터 삭제

### 나이스 학생편성현황 (학년별 xlsx)

예상 열 이름 (유연 매칭):

| 열 | 예시 |
|---|---|
| 학년도 | 2026 |
| 학기 | 1 |
| 학년 | 3 |
| 편제명 | 공통 |
| 개설과목(학점) | 문학(3) |
| 개설강의실 | 문학실 B(3-1) |
| 계열/학년/학과 | 3학년 |
| 반 | 1반 |
| 번호 | 3 |
| 성명 | 홍길동 |

학번 생성 규칙: `학년 + 반(2자리) + 번호(2자리)` → 2학년 1반 3번 = `20103`

| 응시/결시현황표 | 학년, 일차, 교시, 고사실 | 학생 정보, 편성강의실, 비고(결시 등) |

## 개인정보 처리 주의사항

- 학생 성명·학번 등은 **브라우저 메모리에서만** 처리됩니다.
- **외부 서버로 전송되지 않습니다.**
- localStorage 저장·JSON 백업 시 **개인정보가 포함**됩니다.
- 공용 PC 사용 후 반드시 **전체 초기화**하세요.

## 기존 프로그램 대비 설계 (참고 자료 반영)

기존 「고등학교 지필평가 운영 프로그램」은 다음 구조로 동작합니다.

```text
나이스 편성현황 → 배치자료(중앙 DB) → 시험실좌석배치표 / 응시현황표 / 개인별시간표 / 선택과목현황
```

Exam Flow는 이를 다음 단일 데이터 모델로 통합했습니다.

```text
students → studentExamSchedules → examGroups → roomAssignments → seatAssignments → 출력물
```

| 기존 시트/출력물 | Exam Flow 대응 |
|---|---|
| 배치자료 (일차·교시·과목·고사실·좌석) | `roomAssignments` + `seatAssignments` |
| 시험실좌석배치표 | Step 4 좌석배치도 |
| 응시현황표 / 결시현황표 | Step 4 응시현황표 (`attendanceNotes`) |
| 학생개인별시험시간표 | Step 4 개인별 시간표 |
| 선택과목 응시 학생 | Step 4 선택과목 현황표 |
| 시험실배정현황 / 시험실운영현황 | Step 4 시험실배정현황·시험실운영현황 |

## Excel Export 사용 방법

Step 4에서 출력물을 선택한 뒤 **엑셀보내기** 버튼을 클릭하면, 미리보기와 동일한 데이터가 `.xlsx` 파일로 저장됩니다. 모든 처리는 **브라우저에서만** 수행되며 서버로 전송되지 않습니다.

### Export 차단 조건

다음 오류가 있으면 엑셀 생성이 **차단**됩니다.

- 좌석번호 중복
- 시험실 정원 초과
- 미배정 학생

Step 4 배치자료 검토에서 수정한 뒤 다시 시도하세요. **운영 진단** 오류(❌)가 있으면 인쇄·Export가 차단됩니다.

**7종 출력물 + 운영요약** 모두 Excel Export를 지원합니다. Step 4 출력물 카드에 `✅ Excel` 배지가 표시됩니다.
